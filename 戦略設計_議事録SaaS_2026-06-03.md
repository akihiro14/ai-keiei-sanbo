# 戦略設計レポート：議事録SaaS（MeetMemo）
**事業テーマ：会議録音をAIが自動で議事録化し、タスクを担当者に通知するSaaS**
**ターゲット：従業員50名以下の中小企業**

作成日：2026-06-03

---

## PEST分析

### Political（政治・法規制）

#### 追い風となる政策

| 政策・動向 | 概要 | 事業への影響 |
|-----------|------|-------------|
| **AI基本計画（2025年12月閣議決定）** | 「世界で最もAIを開発・活用しやすい国」を国家目標に設定。AI推進法が施行済み | AIツール導入の社会的・政策的後押し。市場全体の信頼醸成 |
| **IT導入補助金2025の拡充** | 中小企業のITツール導入費用に加え「活用支援費用」も補助対象に拡大 | 初期導入コストのハードルが下がり、中小企業への浸透が加速 |
| **DX推進施策（経済産業省）** | 中小企業向けに補助金・専門家派遣・教育プログラムを整備 | 中小企業のIT投資意欲を底上げ。潜在顧客の市場に入る |
| **働き方改革関連法** | 長時間労働規制・多様な働き方の推進が継続 | 会議時間の削減・業務効率化ツールへのニーズが制度的に強化 |

#### 注意すべきリスク

| リスク | 概要 | 事業への影響 |
|--------|------|-------------|
| **AI推進法の規制面** | AI利用時の「透明性確保」「権利保護」の義務化が進む可能性 | 録音・文字起こしデータの取り扱いポリシー強化が求められる |
| **個人情報保護法（改正動向）** | 会議音声・発言内容は個人情報に該当する可能性がある | セキュリティ対策・プライバシーポリシーの整備が必須 |

### Economic（経済）

#### 市場成長データ

| 指標 | 数値 |
|------|------|
| 日本SaaS市場規模（2025年） | 約**122億USD** |
| 日本SaaS市場規模（2035年予測） | 約**381億USD**（CAGR 13.5%） |
| 中小企業（100名以下）のSaaS利用率 | 2024年 38% → 2025年 **48%**（+10pt急上昇） |
| 効率化型SaaS導入企業数の伸び | 年率**20%超** |
| 中小企業の人手不足感（正社員） | **53.4%** が不足（2025年1月・帝国データバンク） |
| 人手不足倒産 | 2024年、**過去最多**を記録 |

#### 事業への示唆

- **人手不足 × 業務効率化** の組み合わせが中小企業の最重要課題。「議事録を人が書く」コストを可視化することで購買動機に直結する
- SaaS利用率が急伸中で「導入すること自体への抵抗感」が低下している
- IT導入補助金の存在が、導入判断のトリガーになりうる

### Social（社会・文化）

#### 会議・議事録に関わる社会変化

| 動向 | データ | 事業への影響 |
|------|--------|-------------|
| **テレワーク定着** | 就業者の24.6%がテレワーカー（2024年度）。ハイブリッド型が定常化 | オンライン会議＝録音しやすい環境が整備済み。議事録AI活用の前提条件が揃っている |
| **議事録作成の負担** | ビジネスパーソンが年間平均**319.6時間**を費やす。67%が負担感を感じている | 課題の認知は広い。ただし「変えられる」と思っていない人が多数 |
| **若手への集中** | 20代は週平均**8.46時間**を議事録作成に費やす | 若手社員の業務体験向上・採用競争力にも訴求できる |
| **出社回帰の動き** | 「意思疎通がしやすい」を理由に出社を好む人が66.4% | 対面会議でも録音→AI議事録は十分に機能する。ユースケースは失われない |
| **少子高齢化・生産年齢人口の減少** | 2030年代にかけて労働力不足が深刻化する見通し | 1人あたり生産性の向上ツールへの長期的需要が底堅い |

### Technological（技術）

#### 技術トレンドと事業機会

| 技術 | 現状・動向 | 事業への活用可能性 |
|------|-----------|------------------|
| **音声認識精度の向上** | 日本語認識精度が最大**98.86%**到達（2025年）。音声認識とLLMの統合が加速 | 文字起こしの精度が実用水準を超え、AIが「使えるツール」として市場に受け入れられる段階 |
| **生成AI（LLM）の進化** | 文字起こし→要約→タスク抽出の完全自動化が実現可能。処理時間は従来の1/5〜1/10 | MeetMemoのコア機能（議事録化・タスク抽出）の技術的実現性が高い |
| **AIエージェントの普及** | 2026年〜「AIエージェント経済」が本格化する見通し | 将来的にMeetMemoをエージェント化し、タスク実行まで自動化する拡張余地あり |
| **API連携の容易化** | Slack/Teams/Chatwork/Zoom等のAPIが整備・標準化されている | 担当者通知・ワークフロー統合を低コストで実装可能 |
| **競合のAI強化** | Microsoft Copilot（Teams標準搭載）、Google Gemini（Meet統合）が本格展開 | 大手プラットフォームへの取り込みリスク。日本語・中小企業特化での差別化が急務 |

## SWOT分析

> ※ PESTのOT（機会・脅威）をSWOT分析に反映しています。

|  | **強み（Strengths）** | **弱み（Weaknesses）** |
|--|----------------------|----------------------|
| **内部環境** | ・**タスク担当者への自動通知**という競合未開拓の機能<br>・日本語特化×日本のビジネス慣行に最適化した設計<br>・ノーコード導入（ボット招待のみ）で IT部門不要<br>・従業員50名以下に絞った**低価格エントリープラン**<br>・Slack / Chatwork / LINE WORKSへのネイティブ通知対応 | ・新規参入のため**認知度・実績がゼロ**<br>・音声認識エンジンは外部API依存（コスト変動リスク）<br>・開発・営業リソースが限定的<br>・Otolio（7,000社実績）等の国内先行者に比べ信頼性で劣る |
| **外部環境** | **機会（Opportunities）** | **脅威（Threats）** |
|  | ・AI議事録ツールの実導入率が**わずか1.4%**→ 巨大な未開拓市場<br>・IT導入補助金2025の拡充で**中小企業の導入障壁が低下**<br>・中小企業SaaS利用率が急伸（38%→48%、+10pt）<br>・人手不足問題（53.4%）が業務効率化需要を後押し<br>・AI基本計画によるAIツール導入の**社会的正当性の向上**<br>・テレワーク定着でオンライン会議録音が常態化 | ・Notta・Fireflies等**グローバルプレイヤーの日本展開加速**<br>・Microsoft Copilot（Teams）・Google Gemini（Meet）の**標準搭載化**<br>・Otolio（18.8%シェア、7,000社）等の国内先行者の存在<br>・AI推進法・個人情報保護法強化による**データ取扱いコスト増**<br>・LLM API料金の変動リスク（外部依存） |

### クロスSWOT（戦略方向性）

| 戦略 | 内容 |
|------|------|
| **SO戦略**（強みで機会を活かす） | AI導入率1.4%の市場に「通知〜タスク完了」一気通貫を訴求。IT補助金を販促フックにした中小企業向けパートナー営業を展開 |
| **WO戦略**（弱みを補いながら機会を取る） | 口コミ・事例公開でゼロから信頼を積み上げ。まず10社の導入事例を作り、それを営業ツールにする |
| **ST戦略**（強みで脅威に対抗する） | 大手プラットフォームが取り込めない「通知＋日本語+中小特化」の深化で差別化を維持 |
| **WT戦略**（弱みと脅威の最小化） | 外部API依存を分散（複数エンジン対応）し、個人情報・セキュリティ対策を先行整備してリスク軽減 |

## 3C分析

### Customer（顧客）

#### ターゲットセグメント：従業員50名以下の中小企業

**デモグラフィック**

| 属性 | 詳細 |
|------|------|
| 企業規模 | 従業員10〜50名（中小企業の約84%がこの規模帯に集中） |
| 意思決定者 | 経営者・役員、または総務・管理部門のリーダー |
| IT環境 | 専任IT担当者がいない企業が多数。クラウドSaaSへの親和性は向上中 |
| 会議頻度 | 週3〜10回。社内会議＋顧客商談が混在 |
| 使用ツール | Zoom / Google Meet / Microsoft Teams、Slack / Chatwork |

**顧客の課題（ペインポイント）**

```
① 議事録作成に時間がかかる（年間319時間）
② 会議で決まったタスクが担当者に伝わらない・忘れられる
③ フォローアップが属人化しており、抜け漏れが多い
④ ITリテラシーが高くなく、複雑なツールを使いこなせない
⑤ 月額数万円以上のツールは予算として通りにくい
```

**購買行動・意思決定フロー**

```
課題認識（「議事録が大変」）
  → 検索・口コミで情報収集
  → 無料トライアルで試す
  → 経営者が費用対効果を判断（月額3,000〜5,000円は即決レベル）
  → 導入・定着（社員が使い続けるかが鍵）
```

**刺さるメッセージ**

> 「会議が終わった瞬間に、担当者のSlackに自動でタスクが届く。議事録は誰も書かなくていい。」

#### 主要競合の詳細

**① Otolio（旧スマート書記）** ← 国内シェアNo.1

| 項目 | 内容 |
|------|------|
| 市場シェア | 18.8%（国内1位） |
| 料金 | 月額 **25,000円〜**（基本料+AIパック） |
| 導入実績 | 7,000社以上（東京都庁・積水化学等） |
| 強み | 高精度・大企業実績・セキュリティ・独自AI学習 |
| 弱み | **価格が高く中小企業（50名以下）には重い**。タスク通知機能は弱い |

**② Notta** ← 日本語精度No.1のグローバルツール

| 項目 | 内容 |
|------|------|
| 料金 | Pro：月額 $13.99（年払い$8.17）、Business：$27.99/シート |
| 強み | 日本語認識精度98.86%、58言語対応、8百万ユーザー |
| 弱み | **タスク担当者通知がない**。CRMはBusinessプラン以上。個人・小チーム向け設計 |

**③ Fireflies.ai** ← グローバルシェアが高い英語特化

| 項目 | 内容 |
|------|------|
| 料金 | Pro：$10/月、Business：$19/月 |
| 強み | 60言語対応、Zapier経由の豊富な連携、CRM統合 |
| 弱み | **日本語対応が弱い**。担当者通知は抽出のみで自動配信は未実装 |

#### 競合の共通弱点（MeetMemoの機会）

1. **全競合が「議事録作成」で止まっており、「タスク通知・追跡」まで担っていない**
2. Otolio は中小企業（50名以下）には高価格すぎる（月2.5万円〜）
3. グローバルツール（Notta/Fireflies）は日本の業務慣行・通知ツールへの最適化が甘い

## 総括：戦略的示唆

### 3分析から見えた最重要アクション

```
1. ターゲットを絞る
   → 「従業員10〜30名・Zoom利用中・Slack通知を使っている」に最初は絞り込む

2. 最速でMVPを出す
   → 機能は3つだけ（録音→議事録→Slack通知）。まず動くものを作る

3. 最初の10社を無料で獲得する
   → 事例とフィードバックが最大の資産。初期ユーザーには永久無料プランで誘致

4. IT補助金パートナーと組む
   → 中小企業診断士・IT補助金代行業者経由での紹介営業が最も効率的な獲得チャネル

5. 差別化軸を死守する
   → 「通知まで届く」「日本語最適化」「月5,000円以下」の3軸はブラさない
```
